Use the screen-casts and notes below to learn "how to" do various common tasks on this site. If you have any comments, suggestions or ideas for new "howtos" please use the feedback tab found on the left hand side of all the pages on this site.

Find Datasets

Key points:

  • Use the search box on the front page.
  • On the results page you can re-order the results and constrain them by publisher, tag, group and/or file type.
  • Clicking on a dataset takes you to its page containing information and resources associated with the dataset (such as data files).
  • Tabular data (as found in CSV files) can be previewed.
  • Use the datasets, publishers and groups items in the main menu at the top to find datasets according to how they are organised in the catalogue.

Work with and Organise Datasets

Key points:

  • Dataset pages contain lots of useful information: title, description, tags, list of resources (CSV files etc) and arbitrary meta-data.
  • Resources can be "explored" via preview of tabular data and some very simple graphing / charting functionality.
  • Datasets can belong to groups, have their history tracked in an activity stream and be linked to related items such as other datasets.
  • Following a dataset means your own dashboard will be populated with activity from the dataset. If you have configured things correctly, the data catalogue will also email when there is new activity.
  • Groups are named sets of datasets and function like named groups of bookmarks (so you can easily find your stuff).
  • It is possible to manage groups so you and your colleagues can collaborate in curating useful groups of datasets.

Publish Data in the Catalogue

Key points:

  • Use open data formats where possible! (CSV good, Excel bad)
  • You must have an "editor role" in a publishing organisation within the data catalogue in order to publish datasets.
  • To create a new dataset click on the "Datasets" item in the main menu and then the "Add Dataset" button on the resulting page.
  • Fill in the form making sure you provide as much information as possible:
    • The title of the dataset is used as the basis for the dataset's URL - a link directly to the dataset.
    • The description should include what the dataset describes, its purpose, notes about methodology and other useful information.
    • Tags are an easy to use mechanism for categorising datasets. If possible re-use the suggested tags.
    • Make sure you select the correct publisher organisation (if you belong to more than one) and set the correct level of visibility (we suggest you start with the dataset being private and only make it public when you're ready). Private datasets are only visible to members of the publishing organisation.
  • Upload various resources remembering to indicate the type of file you're uploading (CSV, PDF, DOC etc).
  • Please provide the raw data for any reports you upload.
  • Remember to add useful metadata!
  • The use "Manage" button on each dataset's page to update its settings.
